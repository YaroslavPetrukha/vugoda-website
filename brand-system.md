# Бренд «Вигода» — витяг для веб-розробки

Джерело: `Вигода_брендбук.pdf` (21 стор., Adobe Illustrator, квітень 2026).

---

## 1. Позиціонування та характер

- **Назва:** ВИГОДА
- **Дескриптор:** «системний девелопмент»
- **Суть:** девелопер, що продає не квадратні метри, а **цілісне середовище** — архітектура + функціональність + інвестиційна доцільність як одна система.
- **Гасло-ядро:** «Системний девелопмент, у якому цінність є результатом точних рішень.»
- **4 цінності:** системність, доцільність, надійність, довгострокова цінність.
- **4 атрибути характеру:** раціональний, впевнений, системний, сучасний.

### Тон комунікації
Чітко · Впевнено · Предметно · Стримано. Без пафосу, без зайвих емоцій, без декоративних слів. Коротко, структуровано, з акцентом на логіку і користь. Ніякого «мрія», «найкращий», «унікальний» — лише факти, цифри, рішення.

Наслідки для сайту:
- Заголовки — декларативні, короткі («Середовище, яке залишається актуальним»), не рекламні.
- Мікрокопі — інструктивне, без маркетингового «ми створюємо чарівні…».
- CTA — функціональні («Переглянути планування», «Отримати розрахунок»), а не емоційні.

## 2. Логотип

Компоненти (з назвами у брендбуці):
1. **Знак/Куб** — ізометричний куб, завжди `#C1F33D` (acid-lime).
2. **Фонова форма «Символ»** — чотири темні кружки-пелюстки за кубом.
3. **Назва** — «вигода» строчним рубаним.
4. **Дескриптор** — «системний девелопмент» під назвою.

Три кольорові версії:

| Фон | Назва | Дескриптор | Куб | Символ (пелюстки) |
|---|---|---|---|---|
| Світлий | `#020A0A` | `#3D3B43` | `#C1F33D` | `#020A0A` |
| Темний (фірмовий `#2F3640`) | `#F5F7FA` | `#A7AFBC` | `#C1F33D` | `#020A0A` |
| Чорний `#000` | `#F5F7FA` | `#A7AFBC` | `#C1F33D` | `#2A3038` |

Правила:
- **Охоронне поле** = висота **великої** літери «В» (cap-height) з фірмового шрифту, з усіх чотирьох боків. У цю зону нічого не ставити. (Помітка: сам логотип написаний строчними, але модуль вимірювання — uppercase.)
- **Мінімальний розмір:** 100 px (екран), 30 мм (друк). Менше — лише знак (куб із пелюстками).
- **Заборонено:** міняти пропорції, композицію, кольори; нахиляти/обертати; масштабувати частини окремо; додавати тіні/ефекти/курсив.

Наслідки для сайту:
- Навбар: версія для темного фону (оскільки фон сайту за замовчуванням темний).
- Фавікон/PWA-icon: «тільки знак».
- Пам'ятати про охоронне поле в `header`/`footer` — лого не має торкатися тексту меню.

## 3. Кольорова палітра

| Роль | HEX | Коментар для UI |
|---|---|---|
| Головний темний (фон) | `#2F3640` | базовий `--bg`, app background |
| Акцентний зелений | `#C1F33D` | `--accent`, тільки для CTA, ключових підсвіток, активних станів. Ніколи — для фону великих блоків або довгого тексту. |
| Світлий основний | `#F5F7FA` | `--text-on-dark`, основний текст на темному |
| Світло-сірий допоміжний | `#A7AFBC` | `--text-muted`, дескриптори, секондарі, плейсхолдери |
| Антрацитовий допоміжний | `#3D3B43` | _використання не прописане в брендбуці._ Рекомендація: `--surface` для карток/панелей, дескриптор на світлому (брендбук вживає його саме для дескриптора логотипа на світлому фоні). |
| Глибокий чорний | `#020A0A` | `--ink`, текст на світлому; чорний фон секцій |

Структура контрасту (з брендбуку прямо): **глибокий темний фон + яскравий акцент**. Це і є головна візуальна ДНК.

Рекомендовані токени для CSS:
```css
:root {
  --bg:          #2F3640;
  --bg-black:    #020A0A;
  --surface:     #3D3B43;
  --accent:      #C1F33D;
  --text:        #F5F7FA;
  --text-muted:  #A7AFBC;
  --ink:         #020A0A;
}
```

Заборони:
- Зелений — не фон широких секцій. Це акцент, не основа.
- Ніяких градієнтів на лініях і ніяких «fancy»-кольорів поза палітрою.

### Контрастність (перевірено калькулятором WCAG 2.1)

| Комбінація | Ratio | Вердикт |
|---|---|---|
| `#F5F7FA` на `#2F3640` | ≈ 10.5 : 1 | AAA — основний текст |
| `#C1F33D` на `#2F3640` | ≈ 8.85 : 1 | AAA — кнопки, цифри, акценти |
| `#C1F33D` на `#020A0A` | ≈ 13.5 : 1 | AAA — акцент на чорному |
| `#A7AFBC` на `#2F3640` | ≈ 5.3 : 1 | AA для normal text (≥14pt). **Не використовувати для дрібного тексту (<14pt), body на цьому кольорі — ризик.** |
| `#C1F33D` на `#F5F7FA` | ≈ 1.2 : 1 | **FAIL.** Зелений на світлому фоні — нечитабельний. Ніколи не використовувати як колір тексту/іконок на білому/світлому. |
| `#020A0A` на `#F5F7FA` | ≈ 20 : 1 | AAA — текст на світлих блоках, якщо такі будуть. |

## 4. Типографіка

- **Сімейство:** **Montserrat** (один шрифт на все) — геометричний гротеск.
- **Ваги:** Bold (H1, H2) · Medium (підзаголовки, акценти) · Regular (body). **Інші ваги (Light, ExtraBold, Black) брендбук не згадує — за замовчуванням не використовувати.**
- **Підтримка:** кирилиця повна (укр. глифи `ґ, є, і, ї`).
- **Character (з брендбуку):** технічна точність + висока читабельність.
- **Джерело:** Google Fonts (SIL OFL, безкоштовно, самохост також OK). Підключення: `@fontsource/montserrat` або `<link>` з Google Fonts.
- **Брендбук не прописує:** letter-spacing/tracking, line-height, абсолютні розміри (px/pt). Шкалу нижче пропоную сам — узгодити з клієнтом.

Масштаб пропозиції (достатньо для старту):

| Роль | Вага | Розмір (desktop) |
|---|---|---|
| H1 / hero | Bold | 56–80 px |
| H2 / section | Bold | 40–48 px |
| H3 / card title | Medium | 24–28 px |
| Lead | Medium | 20 px |
| Body | Regular | 16 px, line-height 1.55 |
| Caption/descriptor | Regular | 13–14 px, колір `--text-muted`, часто uppercase + letter-spacing |

Notes:
- На обкладинках брендбуку: заголовки розділів — Bold у зеленому, підзаголовок — Bold/Medium у білому.
- Lowercase для логотипа та частини заголовків (дисципліна «вигода» з маленької) — не ламати.
- Letter-spacing скромний; ніяких «декоративних» шрифтів як вставок.

## 5. Графічна система

Ключова мова: **лінійна ізометрія + каркасні структури** (куб є основним модулем).

Три сходинки складності:
1. **Базовий модуль** — один ізометричний куб у каркасі.
2. **Структурна група** — 2–3 куби, що з'єднуються гранями.
3. **Ізометрична сітка (патерн)** — повторення для фонів, hero, розділових секцій.

Параметри ліній:
- Stroke: **0.5–1 pt** (≈1–1.5 px на екрані). Максимум 1.5 pt.
- Колір: `#A7AFBC`, `#F5F7FA`, або `#C1F33D`.
- Opacity: **5–60 %** (саме так у брендбуці — ледь помітні на фоні).
- Стиль: Butt Cap, Miter Join. Лише прямі лінії.

Дозволено:
- Як фоновий патерн.
- Накладання на фото в режимах Overlay / Screen.
- Схеми та ілюстрації (наприклад, поверхи, зонування).

Заборонено:
- Перекривати елементи логотипу.
- Кольорові градієнти на лініях.
- Лінії товщі за 1.5 pt.

Наслідки для сайту:
- Hero — темний фон + ледь видний ізометричний патерн в opacity 10–20 %.
- Секції-розділювачі — одиничні куби як decoration-елементи (SVG, inline).
- Підійде `three.js` лише якщо тримати візуал плоским у стилі каркасу (не реалістичні рендери).

## 6. Чек-лист «як це має виглядати на сайті»

**DO**
- Темний фон `#2F3640` за замовчуванням, чорні секції `#020A0A` — для «зваженого» акценту.
- Зелений `#C1F33D` — тільки на кнопках, посиланнях у тексті, активному табі, чіп-тезі, цифрі-лічильнику. Точкові дози.
- Один шрифт (Montserrat), три ваги.
- Велика негативна площа (air). З брендбуку видно: багато порожнечі навколо заголовків.
- Структура сторінки = ієрархія сіткою (3–4 колонки, чіткі поділи тонкими лініями).
- Ізометричний куб як повторюваний візуальний якір.

**DON'T**
- Не змішувати з іншими яскравими кольорами. Палітра закрита.
- Не використовувати глоу, тіні, блюр, неонові ефекти на логотипі та кубі.
- Не додавати курсив, декоративні шрифти, рукописні акценти.
- Не робити зелений фоном широких блоків — тільки акцент.
- Не використовувати стокові «смайлові» ілюстрації. Тільки каркасна ізометрія, креслення, реальні фото архітектури з патерном-оверлеєм.
- Не ламати охоронне поле логотипа.

## 7. Прогалини брендбуку (критично для веб-проєкту)

**TOC на стор. 2 обіцяє розділ «20 Сітка та композиція», але фактична стор. 20 — це «Графічні елементи» (параметри ліній + бібліотека форм-кубів). Справжньої layout-сітки, колонок, gutters, breakpoints, spacing scale у брендбуці немає.** Це ключовий пробіл — його треба закрити на старті дизайн-системи сайту, інакше кожна секція буде «на око».

Повний список відсутнього:

| Категорія | Статус | Що робити |
|---|---|---|
| Layout-сітка (колонки, gutters) | відсутня | Запропонувати 12-col на desktop, 4-col mobile, gutter 24/16 px |
| Spacing scale (rhythm) | відсутня | Запропонувати 4/8/12/16/24/32/48/64/96 px |
| Breakpoints | відсутні | Стандарт: 640/768/1024/1280/1536 |
| Кольори станів (success/warning/error) | відсутні | Запропонувати стриману трійку або вирішувати контуром + `--accent` |
| Іконки | відсутні | Побудувати outline-only, stroke 1.5 px, кутові з'єднання (Miter Join) — у єдиному стилі з ізометрією |
| Фотостиль | відсутній | Архітектурні кадри, холодний ББ, накладання ізометричної сітки Overlay 10–20 % |
| UI-компоненти | відсутні | Проєктуємо дизайн-систему на базі правил цього документа |
| Motion/animation | відсутні | Узгодити з клієнтом: повільна, стримана, ease-out; без «bounce» |
| Приклади копірайту | відсутні | Написати voice-guide зі зразками (headlines, CTA, error-message) |
| Letter-spacing/line-height | відсутні | Запропонувати: body 1.55, H1 1.1, caps tracking 0.04em |
| Light-mode сторінки? | не описано | Брендбук весь дарк. Перевірити, чи взагалі потрібен light-mode |
| Social-media / favicon-шаблони | відсутні | Окремо створити |

## 8. Потрібні вихідні асети (запитати у клієнта)

- **Логотип у векторі** (.ai, .svg або обидва) для трьох версій фонів. З PDF брендбуку я можу витягти векторний контур, але надійніше мати оригінал.
- **Шрифтовий файл Montserrat** — Google Fonts (OFL) бере всі проблеми з собою, додаткового ліцензування не треба; але якщо клієнт уже купив/має конкретну версію — перевірити.
- **Фотобанк** — архітектурні рендери, фото будівельних майданчиків, планування. Папка `Рендера/` у проекті вже існує — перевірити вміст.
- **Копірайт** — тексти про компанію, місію, проєкти, послуги.
- **Список об'єктів** (ЖК/проєктів) — структура, стадії, характеристики.
