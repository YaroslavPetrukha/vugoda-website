3. **Західний регіон — №1 по зростанню.** Біженці з Києва/Сходу розігріли попит.
4. **Менше нових ЖК.** Вибір звужується → конкуренція за якісного покупця зростає.
5. **Інвестиційний нарратив.** Зростання цін + слабка гривня = квартира як store-of-value.
6. **єОселя домінує.** Якщо об'єкт не акредитований — втрачаємо ~17% попиту.

## 7. Рекомендації для структури сайту

### Тип сайту
**Корпоративний багатосторінковий сайт забудовника** — не лендінг і не каталог ЖК. Центр тяжіння — **Вигода як компанія** (процес, методологія, гарантії, портфель). Окремі ЖК — вторинні сутності, які живуть на своїх сайтах або мінімальних внутрішніх сторінках.

### Архітектура портфеля (патерн "hub → spokes")

**Правило для сторінки ЖК:**
1. **Якщо у ЖК є власний сайт/лендінг** (як Lakeview):
   - На корпоративному сайті — коротка картка об'єкта (1 екран: назва, клас, стадія, 1–2 факти, render placeholder) + **кнопка/редирект** на зовнішній сайт ЖК.
   - Альтернатива: внутрішня "превʼю-сторінка" (slug `/zhk/{slug}`) з аналогічним мінімальним вмістом і тією ж кнопкою-редиректом на зовнішній сайт.
   - Рішення (hard-redirect vs превʼю-сторінка) — **брейншторм на етапі планування** (див. п. нижче).

2. **Якщо у ЖК немає власного сайту**:
   - Повноцінна внутрішня сторінка `/zhk/{slug}` у стилі брендбуку Вигоди (не в стилі проекту).
   - Контентна модель має тримати різні стадії: "у проектуванні" / "у погодженні" / "будується" / "здано".
   - Для pipeline-проектів без рендерів — нейтральний візуал (ізометричний куб-патерн з брендбуку, схема стадій, блок "скоро деталі").

**Відкриті питання для планування** (на стадії ТЗ):
- Hard-redirect на зовнішній сайт vs внутрішня превʼю-сторінка з CTA "Перейти на сайт ЖК" — що краще для SEO, для UX, для швидкості лідогенерації?
- Чи тримаємо базові метадані ЖК (ціна, стадія) у CMS на стороні Вигоди для внутрішнього каталогу, чи все на стороні проекту?
- Як уникнути канібалізації SEO (корп. сайт vs сайт ЖК цілять в одні запити)?

### Must-have сторінки (MVP корпоративного сайту)
1. **Головна** — герой у стилі брендбуку (ізометрія, темний фон, точкова лайм-акцент) + канонічний меседж + блок "Чим ми займаємось" + огляд портфеля + CTA
2. **Про компанію** — юрособа, ліцензія, принцип роботи (системний девелопмент), історія, **без фото команди**; факт-блок (ЄДРПОУ, ліцензія, клас наслідків об'єктів)
3. **Портфель / Наші проекти** (хаб-сторінка) — список усіх ЖК з фільтрами по стадії: `Будується` / `У pipeline` / `Здано` → кожен елемент веде на картку ЖК (внутрішню або зовнішню)
4. **Картка ЖК** (шаблон, за патерном вище) — слаги `/zhk/lakeview`, `/zhk/{project-2}`, ...
5. **Як ми будуємо** — технологія (монолітно-каркас), безпека (укриття, СС3), якість матеріалів, контроль будівництва; **без "наша команда"**, акцент на процес і сертифікати
6. **Умови купівлі** — типові умови, розтермінування, знижки, переуступка, єОселя (де застосовно); підкреслити, що конкретика — на сторінках ЖК
7. **Для інвесторів** — спеціальна сторінка: дохідна нерухомість (коли буде), переуступка, юр. захист, часті питання
8. **Контакти** — карта, форма, `vygoda.sales@gmail.com`, телефон (після уточнення), месенджери; реквізити юрособи

### Принципи контент-архітектури
- **Масштабованість під N проектів** — шаблон картки ЖК має тримати 5, 10, 20 об'єктів без редизайну. Не хардкодимо конкретні назви поточного портфеля.
- **Стан об'єкта — перший клас** — кожна картка має явний атрибут "стадія" (pipeline/будується/здано); фільтрація і сортування по стадії; різний шаблон презентації залежно від стадії.
- **Pipeline без рендерів — окремий UX-стан** — не показуємо порожні картки з плейсхолдерами "фото скоро"; натомість — ізометричний куб-блок + текстовий статус + "підписатись на оновлення".

### Tone of voice (з брендбуку)
**Чітко · Впевнено · Предметно · Стримано.** Без пафосу, без зайвих емоцій, без декоративних слів.

Практичні наслідки:
- **Ніякого словника:** "мрія", "найкращий", "унікальний", "преміальний стиль життя", "ваша історія починається тут"
- **Заголовки декларативні, короткі** — стан справ, не рекламний заклик
- **Мікрокопі інструктивне** — "Оберіть поверх", "Залиште номер — зателефонуємо сьогодні"
- **CTA функціональні:** "Переглянути планування", "Отримати розрахунок", "Записатись на огляд" (не "Почніть свою мрію")
- **Факти над обіцянками:** ЄДРПОУ, ліцензія від 27.12.2019, клас наслідків СС3, термін здачі — конкретна дата
- **Військовий контекст:** стримано, підтримуюче, без експлуатації болю; укриття і безпека — як технічна специфікація, не як емоційний тригер

### Важлива чесність: портфель у формуванні
На корпоративному сайті не імітуємо "20 років досвіду" і "тисячі щасливих мешканців". Поточна реальність:
- 0 зданих об'єктів, 1 активний (Lakeview), 4 у pipeline
- Компанія молода, бренд тільки формується
- Ліцензія з 2019 року, юридично чистий трек

**Меседж-лінія на цьому етапі:** замість "ми вже збудували багато" — "ми ретельно підходимо до кожного рішення, і ось як це виглядає на практиці". Показуємо процес, методологію, прозорість — речі, які не потребують зданого портфеля, але створюють довіру. Брендбукова лінія "системний девелопмент" органічно покриває цей розрив.
