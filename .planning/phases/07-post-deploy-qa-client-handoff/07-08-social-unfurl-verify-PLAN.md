    Per D-03 absorption pairing (UAT-3 procedure):

    **Setup:**
    1. `mkdir -p .planning/phases/07-post-deploy-qa-client-handoff/unfurls`
    2. Confirm the deployed URL: `https://{account}.github.io/vugoda-website/` (Phase 6 D-37 outcome — likely yaroslavpetrukha).

    **Per-platform procedure (3 captures):**

    **1. Viber:**
    1. Open Viber (mobile or desktop) → open ANY chat (yourself, a test contact, or a dedicated test conversation — NOT a public group, to avoid spamming).
    2. Paste `https://{account}.github.io/vugoda-website/` in the message input.
    3. Wait 2-5 seconds for Viber to fetch and render the unfurl card BEFORE sending.
    4. The unfurl card should appear ABOVE the input area showing: og:image (or thumbnail) + og:title + og:description.
    5. Capture the screen showing the unfurl card. On macOS use Cmd+Shift+4 to region-capture, on iPhone press the Side+Volume-Up buttons.
    6. Save as `.planning/phases/07-post-deploy-qa-client-handoff/unfurls/viber.png`. (Don't actually send the message if avoidable.)

    **2. Telegram:**
    1. Open Telegram (desktop or web; mobile also works).
    2. Open any chat (Saved Messages is ideal — it's your own DM-to-self).
    3. Paste `https://{account}.github.io/vugoda-website/` in the input.
    4. Wait for the «Page preview» preview block to appear ABOVE the input (Telegram's UI for OG cards).
    5. Capture the screen showing the preview block.
    6. Save as `.planning/phases/07-post-deploy-qa-client-handoff/unfurls/telegram.png`.
    7. **If preview is missing/stale:** Telegram caches aggressively. Force-refresh via:
       - Append `?v=2` to the URL: `https://{account}.github.io/vugoda-website/?v=2` (Telegram treats this as a new URL)
       - OR forward the URL to @WebpageBot in Telegram and ask «refresh» — bot triggers re-scrape
       - Then re-paste and capture.

    **3. Slack:**
    1. Open any Slack workspace where you have permission to post.
    2. Open any channel or DM (Saved/Notes channel is best — Slack's drafts area).
    3. Paste `https://{account}.github.io/vugoda-website/` in the input.
    4. Wait for the unfurl preview to appear BELOW the input (Slack's standard OG render).
    5. Capture the screen showing the preview.
    6. Save as `.planning/phases/07-post-deploy-qa-client-handoff/unfurls/slack.png`.
    7. **If preview is missing/stale:** Slack caches per workspace. Try:
       - Append `?v=2` query param
       - OR ask workspace admin to use `/unfurl` command with the URL
       - Then re-paste and capture.

    **Pass criteria for each platform's unfurl:**
    - [ ] **og:image visible:** the 1200×630 hero-cube render is present (not a broken image, not a placeholder/fallback)
    - [ ] **Title correct:** displays «ВИГОДА — Системний девелопмент» (or close — some platforms truncate)
    - [ ] **Description visible:** displays the og:description (or first ~200 chars of it). Expected text starts with «Системний девелопмент, у якому цінність…»
    - [ ] **No URL-as-fallback:** the unfurl is NOT just rendering the URL string with no card

    If any platform shows broken image or no card:
    - Verify the OG meta in deployed index.html: `curl -s https://{account}.github.io/vugoda-website/ | grep -E 'og:(title|image|description|url)'` — all 4 should be present.
    - Verify og-image.png loads: `curl -I https://{account}.github.io/vugoda-website/og-image.png` — should return 200 + Content-Type: image/png.
    - If the image 404s, the prebuild OG-image step (Phase 6 build-og-image.mjs) didn't run on the last deploy → trigger a fresh push to redeploy.
